# Attendance_CSV

CSV から名簿を読み込んで、Web UI 上で出欠をチェック・班分けできるシンプルな Flask アプリ。
SQLite に保存するのでサーバーは 1 プロセスで完結し、追加のミドルウェアは不要です。

## 主な機能

- **CSV 名簿の取り込み**（`名前` カラム必須、`班` は専用カラム、その他は自動で表示）
- **文字コード自動判定**（UTF-8 / UTF-8 BOM / Shift-JIS(cp932)）
- **3 状態の出欠管理**: 未確認（初期値）/ 出席 / 欠席
  - ドロップダウン (`<select>`) で切替。スマホではネイティブピッカーが開く
  - 状態は色（緑＝出席 / 赤＝欠席 / 無色＝未確認）で一目で分かる
  - 変更は非同期 (fetch) で即保存、出席 / 欠席 / 未確認 / 全体のカウンタが更新される
- **班 (group) 管理**
  - 班管理ページで班の追加・名前変更、各メンバーの班割り当てができる
  - 一覧画面では班ごとにフィルタできる
- **検索 & フィルタ**（一覧画面）
  - 名前や任意項目を含むキーワード検索
  - 出欠状態でのフィルタ
  - 班、および一意値が 2〜15 個の項目（学年・所属など）に自動でフィルタを追加
- **全員リセット**: 出欠を一括で未確認に戻す（root のみ）
- **2 ロールのアクセス制御**: `root`（管理者）/ `user`（一般）

## セットアップ

```bash
# 1. 仮想環境を作成
python -m venv .venv

# 2. 仮想環境を有効化
# Windows (PowerShell)
.venv\Scripts\Activate.ps1
# Windows (cmd)
.venv\Scripts\activate.bat
# macOS / Linux
source .venv/bin/activate

# 3. 依存ライブラリをインストール
pip install -r requirements.txt

# 4. .env を作成してパスワードを設定
copy .env.example .env       # Windows
# cp .env.example .env        # macOS / Linux
# .env を編集して ROOT_PASSWORD と USER_PASSWORD に「別々の」値を記入

# 5. 起動
python app.py
```

ブラウザで <http://localhost:5000> にアクセス。

> `attendance.db` は初回起動時に自動生成されます。`ROOT_PASSWORD` と `USER_PASSWORD`
> が未設定、または同じ値だと起動時にエラーになります。

## 環境変数 (.env)

| 変数 | 必須 | 説明 |
|---|---|---|
| `ROOT_PASSWORD` | ✓ | 管理者ログイン用パスワード |
| `USER_PASSWORD` | ✓ | 一般ユーザーログイン用パスワード（root と別の値） |
| `FLASK_SECRET_KEY` | – | セッション署名鍵。未設定なら起動時に自動生成し DB に保存 |

## 使い方

1. ログイン画面でパスワードを入力（入力欄は 1 つ。入力値で root / user を自動判定）
2. **(root のみ)** 「CSV取込」から名簿 CSV をアップロード
   - 1 行目はヘッダー、`名前` カラムを必ず含めてください
   - `班` カラムがあれば班として取り込まれます（任意）
   - その他のカラム（学年、所属、メモなど）は自動で一覧に表示されます
   - 取り込んだ直後は全員「未確認」状態になります（既存データは上書き）
3. 「出欠一覧」で各メンバーのドロップダウンから 未確認 / 出席 / 欠席 を選択
4. 上部の検索ボックスやフィルタで対象を絞り込み（班・一意値が少ない項目は自動でフィルタ化）
5. 「班管理」で班の追加・名前変更や、各メンバーの班割り当てを変更
6. **(root のみ)** 「全員を未確認に戻す」で出欠状態を一括クリア

## ロールと権限

| 操作 | root | user |
|---|---|---|
| 出欠一覧の閲覧・検索・フィルタ | ✓ | ✓ |
| 各メンバーの出欠状態の変更 | ✓ | ✓ |
| 班の割り当て・追加・名前変更 | ✓ | ✓ |
| CSV アップロード（名簿の入れ替え） | ✓ | ✗ |
| 全員リセット | ✓ | ✗ |

未ログインで保護ページを開くとログイン画面へリダイレクト、API は 401 を返します。
user で root 専用の操作を試みると、画面操作はリダイレクト＋メッセージ、API は 403 になります。

## CSV サンプル

```csv
名前,班,学年,所属
山田太郎,1,3,A組
佐藤花子,2,3,B組
鈴木一郎,サブ1,2,A組
```

- `名前` … 必須。空の行はスキップされます。
- `班` … 任意の専用カラム。後から班管理ページで変更できます。
- それ以外の列 … 自動検出され、一覧に表示・フィルタ対象になります。

## 出欠ステータス

| 値 | 意味 | 表示 |
|---|---|---|
| 0 | 未確認 | 無色（初期値） |
| 1 | 出席   | 緑 |
| 2 | 欠席   | 赤 |

## API

すべてログイン必須。レスポンスは JSON。

| メソッド | パス | 用途 | 権限 |
|---|---|---|---|
| POST | `/api/set/<id>` | 指定メンバーの出欠を設定（`status=0\|1\|2`） | root / user |
| POST | `/api/reset` | 全員を未確認 (status=0) に戻す | root |
| POST | `/api/set_group/<id>` | 指定メンバーの班を設定（`group_num=<班名>`） | root / user |
| POST | `/api/groups/add` | 班を追加（`name=<班名>`） | root / user |
| POST | `/api/groups/rename` | 班名を変更（`old_name`, `new_name`） | root / user |

- `/api/set` と `/api/reset` のレスポンスには 出席 / 欠席 / 未確認 / 全体 のカウントが含まれます。
- 班名は最大 50 文字。重複や空文字はエラー (400) になります。

## 補足・制約

- アップロード上限は 5MB。超えると「ファイルが大きすぎます」エラー (413) になります。
- パスワード照合は `hmac.compare_digest` を使用（タイミング攻撃対策）。
- 開発用に `app.run`（`127.0.0.1:5000`, `debug=False`）で起動します。
  公開する場合は gunicorn など本番用 WSGI サーバーの利用を推奨します。
